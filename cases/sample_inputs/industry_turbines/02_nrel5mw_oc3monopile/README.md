# NREL 5MW on the OC3 monopile substructure

OC3 monopile substructure, cantilever clamped at TowerBsHt = +10 m above MSL (treats pile as rigid extension below; for combined pile + tower physics use Tower.from_elastodyn_with_subdyn).

## Files

| File                              | Purpose                          |
| --------------------------------- | -------------------------------- |
| `02_nrel5mw_oc3monopile.bmi`                | Main BModes-format input         |
| `02_nrel5mw_oc3monopile_sec_props.dat`      | Distributed tower section data   |

## How to run

```python
from pybmodes.models import Tower
tower = Tower("cases/sample_inputs/industry_turbines/02_nrel5mw_oc3monopile/02_nrel5mw_oc3monopile.bmi")
modal = tower.run(n_modes=8)
print(modal.frequencies[:4])
```

## pyBmodes 1st-FA tower-bending frequency

- pyBmodes (cantilever from TowerBsHt, this BMI): **0.3511 Hz**
- Published reference: ~ 0.30 Hz
- Reference source: Jonkman & Musial 2010 OC3 Phase II — approximation; flexible-pile system 1st-FA at ~ 0.275 Hz

The pyBmodes value is for a cantilever clamped at TowerBsHt with the RNA lumped at the tower top. Where the published reference is for the flexible-pile + tower combined-cantilever system (most monopile cases), the cantilever-only frequency is **higher** by 5–25 % depending on pile length and stiffness. For the combined-cantilever physics use:

```python
from pybmodes.models import Tower
tower = Tower.from_elastodyn_with_subdyn(
    "path/to/<turbine>_ElastoDyn.dat",
    "path/to/<turbine>_SubDyn.dat",
)
```
